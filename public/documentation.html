Introducción

Manvi es un asistente personal por WhatsApp usando Twilio. Escribes en español (o mezclado); el bot entiende la intención con Gemini (y respaldos), guarda datos en Supabase y responde por el mismo canal.

Es un proyecto autohospedado: tú despliegas el servidor (por ejemplo Railway o Render), configuras las APIs y las variables de entorno. Los proveedores externos que usa este código son: Twilio (mensajes), Supabase (PostgreSQL), Google AI (Gemini), Groq y OpenRouter (IA opcional en cascada), Tavily y Serper (búsqueda web).

Para quién es

Diseñado para un solo dueño. No es multi‑inquilino. Recordatorios, contactos y eventos quedan asociados al número que configures en MY_PHONE_NUMBER y a tu cuenta de Twilio/Supabase.

Capacidades principales

⏰ Recordatorios
Puntuales con hora y, si aplica, fecha. Tiempos vagos (mañana, tarde, noche) los resuelve la IA según reglas del prompt.
🔁 Rutinas diarias
Misma hora todos los días hasta que las borres.
📅 Semanales y mensuales
Día de la semana o día del mes con hora fija.
✏️ Editar
Corrige un recordatorio pendiente con un mensaje de seguimiento.
🧠 Memoria
Los últimos 4 intercambios (usuario + bot) se envían a la IA como contexto.
🎂 Eventos
Cumpleaños y fechas especiales con avisos programados por el planificador.
📇 Contactos
Agenda y reenvío de mensajes a contactos guardados.
🔍 Búsqueda web
Tavily primero; si falla o sin cupo, Serper. Resumen orientado a WhatsApp.
💬 Chat
Preguntas generales por la misma cascada de modelos (cuota diaria limitada en código).
🗑 Eliminar
Borra recordatorios, rutinas, recurrentes o eventos por nombre aproximado.
🎤 Medios
Solo se procesa el texto de Body. Notas de voz o adjuntos sin texto suelen ignorarse (sin transcripción integrada).
🩺 Panel de estado
Uso de APIs, jobs en segundo plano y health en /status y /api/status.
⚡️ Auto‑ping
Con PUBLIC_URL, el servidor se hace ping a /api/ping para reducir que el PaaS duerma el proceso.

Inicio rápido

1. Clonar el repositorio

git clone https://github.com/sergioesv/whatsapp-reminder-bot.git
cd whatsapp-reminder-bot
npm install

2. Variables de entorno

cp .env.example .env

Edita .env con todas las claves necesarias. Lista detallada en Variables de entorno.

3. Base de datos

En Supabase, SQL Editor: ejecuta el esquema de Base de datos. Si tu tabla personal_reminders es antigua, aplica también sql/personal_reminders_add_group_name.sql.

4. Ejecutar en local

npm run dev

Servidor en http://localhost:3000. Panel en http://localhost:3000/status.

5. Pruebas

node test.js

Comprueba integración contra tu proyecto Supabase (requiere .env válido). No envía WhatsApp reales en los tests habituales.

Base de datos (Supabase)

Ejecuta este SQL una vez en el SQL Editor de Supabase. Crea las tablas que usa el código (contacts, personal_reminders con group_name, daily_routines, recurring_tasks, special_events, interaction_logs, api_usage, system_jobs).

Si ya tenías personal_reminders sin group_name, corre además el script del repo: sql/personal_reminders_add_group_name.sql.

SQL-- Contacts address book
CREATE TABLE contacts (
  id    SERIAL PRIMARY KEY,
  name  VARCHAR(50) NOT NULL UNIQUE,
  phone VARCHAR(20) NOT NULL
);

-- One-off reminders
CREATE TABLE personal_reminders (
  id            SERIAL PRIMARY KEY,
  phone         VARCHAR(20) NOT NULL,
  message       TEXT NOT NULL,
  reminder_time TIMESTAMP WITH TIME ZONE NOT NULL,
  group_name    VARCHAR(50),
  status        VARCHAR(20) DEFAULT 'pending',
  created_at    TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Daily routines (time only, fires every day)
CREATE TABLE daily_routines (
  id              SERIAL PRIMARY KEY,
  phone           VARCHAR(20) NOT NULL,
  task_name       TEXT NOT NULL,
  reminder_time   TIME NOT NULL,
  is_active       BOOLEAN DEFAULT TRUE,
  last_fired_date DATE
);

-- Weekly and monthly recurring tasks
CREATE TABLE recurring_tasks (
  id               BIGSERIAL PRIMARY KEY,
  phone            VARCHAR(20) NOT NULL,
  task_name        TEXT NOT NULL,
  reminder_time    TIME NOT NULL,
  recurrence_type  TEXT NOT NULL CHECK (recurrence_type IN ('weekly', 'monthly')),
  day_of_week      INTEGER,
  day_of_month     INTEGER,
  is_active        BOOLEAN DEFAULT TRUE,
  last_fired_date  DATE,
  created_at       TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Birthdays and special events
CREATE TABLE special_events (
  id          SERIAL PRIMARY KEY,
  phone       VARCHAR(20) NOT NULL,
  event_type  VARCHAR(50),
  person_name VARCHAR(100),
  event_date  DATE NOT NULL,
  created_at  TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- All incoming messages and bot responses (also powers conversational memory)
CREATE TABLE interaction_logs (
  id           SERIAL PRIMARY KEY,
  sender_name  VARCHAR(50),
  sender_phone VARCHAR(20) NOT NULL,
  message      TEXT NOT NULL,
  bot_response TEXT,
  created_at   TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Daily API usage tracking (stores uptime history)
CREATE TABLE api_usage (
  usage_date       DATE PRIMARY KEY,
  gemini_count     INT DEFAULT 0,
  groq_count       INT DEFAULT 0,
  openrouter_count INT DEFAULT 0,
  tavily_count     INT DEFAULT 0,
  serper_count     INT DEFAULT 0,
  error_count      INT DEFAULT 0
);

-- v1.1.1: Background job health tracking
CREATE TABLE system_jobs (
    job_name TEXT PRIMARY KEY,
    last_fired TIMESTAMPTZ,
    status TEXT DEFAULT 'active'
);

INSERT INTO system_jobs (job_name, status)
VALUES 
    ('Reminder Dispatch', 'active'),
    ('Routine Dispatch', 'active'),
    ('Recurring Task Dispatch', 'active'),
    ('Event Alert', 'active')
ON CONFLICT (job_name) DO NOTHING;

Twilio y WhatsApp

El webhook de este proyecto es HTTP POST en la ruta /webhook (mismo servidor que sirve la web). Twilio envía el cuerpo del formulario con al menos Body, From y To.

  1. En Consola Twilio, abre tu número de WhatsApp o el Sandbox de WhatsApp.
  2. Configura When a message comes in con la URL https://TU_DOMINIO_PUBLICO/webhook y método HTTP POST.
  3. Asegúrate de que las variables TWILIO_ACCOUNT_SID, TWILIO_AUTH_TOKEN y TWILIO_WHATSAPP_NUMBER coincidan con ese entorno.
  4. El bot responde con el mismo canal usando src/sendMessage.js (API de mensajes de Twilio).
No es la API de Meta Graph

Esta rama del proyecto no usa PHONE_NUMBER_ID ni ACCESS_TOKEN de Meta. Si tu documentación antigua mencionaba Meta, sustitúyelo por la configuración de Twilio anterior.

Recordatorios puntuales

Escribe en lenguaje natural. La IA (CERO en el prompt) devuelve JSON con hora y, si aplica, fecha; el servidor guarda reminder_time con zona America/Bogota y el planificador envía el mensaje por Twilio cuando toca.

Ejemplos

Recuérdame llamar al banco a las 3 de la tarde
En 20 minutos recuérdame revisar el horno
El 5 de abril tengo entrevista, recuérdame a las 9:00

Cómo se resuelve la hora

  • Relativo (“en 20 minutos”): la IA calcula HH:MM:SS según la hora actual en Colombia indicada en el prompt.
  • Fecha + hora: se guarda un instante concreto en base de datos.
  • Solo hora: se usa el día actual en Bogotá; si esa hora ya pasó, se mueve al día siguiente (buildReminderDate).
  • Fecha en el JSON sin evidencia en el texto: el servidor puede descartar fechas muy lejanas para evitar alucinaciones; conviene nombrar el día en el mensaje.
Zona horaria

Recordatorios, rutinas y eventos especiales usan America/Bogota (COT, −05:00) en el código del planificador y en buildReminderDate.

Rutinas diarias

Misma hora de reloj todos los días, sin fecha de fin. Van en la tabla daily_routines.

Para repeticiones cada X minutos dentro de un tramo horario, usa recordatorios por intervalo.

Recuérdame beber agua todos los días a las 10:00
Rutina diaria revisar el correo a las 9:00

Para quitarla: “Elimina la rutina de beber agua” (o similar).

Recordatorios por intervalo

Varias alertas espaciadas (mínimo 5 minutos entre sí) dentro de una ventana en horas (por defecto 8 h). Se insertan varias filas en personal_reminders con group_name = 'interval'.

Ejemplos

Recuérdame cada 30 minutos tomar agua
Cada hora recuérdame estirar durante las próximas 4 horas

Funcionamiento

  • intervalMinutes — frecuencia. Mínimo 5.
  • durationHours — duración de la ventana; por defecto 8 si no viene en el JSON.
  • El cron de recordatorios (cada minuto) ya procesa esas filas igual que las puntuales.

Verlos

Rutina vs intervalo

La rutina es siempre a la misma hora civil cada día. El intervalo es muchas notas dentro de unas horas y termina.

Cumpleaños y eventos

Tabla special_events. El año puede servir para la primera carga; los avisos recurrentes los maneja el job de alertas.

El 22 de mayo es el cumpleaños de Ana
Guarda el aniversario de Luis el 3 de abril

El planificador incluye un job (Event Alert) que refuerza alertas a las 08:30 hora Colombia según la lógica en src/scheduler.js.

Contactos y mensajes

Tabla contacts. Solo el dueño (MY_PHONE_NUMBER) puede guardar o listar agenda completa según la lógica actual.

# Guardar
Guarda a mamá como 573001234567

# Mensaje por nombre
Dile a mamá que llego en 10 minutos

Número con prefijo país; el código normaliza quitando caracteres no numéricos al guardar.

Búsqueda web

src/search.js: primero Tavily, si no hay resultado o clave, Serper. El resumen final lo genera de nuevo la cascada de IA sobre los resultados.

¿Quién ganó el último torneo?
Clima en Bogotá hoy
Noticias recientes de …

Los límites y avisos de cupo bajo están en src/usage.js y pueden notificarse por WhatsApp al dueño.

Recordatorios semanales y mensuales

Tabla recurring_tasks; cron separado en el planificador.

Semanal

Cada martes a las 20:00 recuérdame sacar la basura

Mensual

El día 1 de cada mes a las 9:00 recuérdame pagar el arriendo
Fin de mes

Si pides día 31 y el mes es más corto, el scheduler puede ajustar al último día del mes (comportamiento descrito en comentarios del proyecto).

Editar recordatorio

Intent edit_task: localiza un recordatorio pendiente por texto parecido, borra la fila y crea otra con la nueva hora.

Recuérdame comprar huevos a las 17:00
Mejor a las 18:00 el de los huevos
Solo pendientes puntuales

No sustituye editar rutinas diarias, recurrentes o eventos en bloque; ahí conviene borrar y crear de nuevo.

Memoria conversacional

Se leen hasta 4 pares (mensaje / respuesta) de interaction_logs para el mismo sender_phone antes de llamar a la IA.

¿Quién ganó …?
¿Y el capitán del equipo?

El historial no se mezcla entre números distintos.

Notas de voz, imágenes y archivos

El webhook solo usa req.body.Body como texto. Twilio puede enviar adjuntos (NumMedia, MediaUrl0…); este código no los descarga ni transcribe. Si mandas solo nota de voz, Body suele ir vacío y el servidor termina la petición sin contestar.

Para soportar voz hace falta integrar un servicio STT y ampliar el manejador del webhook.

Despliegue (Railway, Render u otro PaaS)

Necesitas un proceso Node accesible por HTTPS para que Twilio pueda hacer POST a /webhook.

Railway (ejemplo)

  1. Conecta el repo de GitHub al proyecto en Railway.
  2. Comando de arranque: npm start (usa src/server.js según package.json).
  3. Define todas las variables de la tabla Variables de entorno en el panel (no subas .env al repo).
  4. Asigna dominio público y copia la URL base a Twilio y a PUBLIC_URL.

Render (alternativa)

  1. Nuevo Web Service desde el mismo repositorio.
  2. Build: npm install (o el predeterminado). Start: npm start.
  3. Mismas variables de entorno que en Railway.
Instancias que duermen

En planes gratuitos el contenedor puede apagarse sin tráfico. Usa PUBLIC_URL y/o un monitor externo que golpee /api/ping.

Mantener el servicio activo y vigilancia

A: Auto‑ping (en el código)

Si defines PUBLIC_URL con la URL pública de tu app, el servidor hace GET a /api/ping cada 10 minutos (axios).

PUBLIC_URL=https://tu-servicio.up.railway.app

B: Monitor externo (UptimeRobot u otro)

UptimeRobot u homólogo: monitor HTTPS cada unos minutos hacia:

https://TU_DOMINIO/api/ping

Respuesta 200 si Supabase responde; 500 si hay error consultando la base (ver src/server.js).

Pruebas locales

node test.js usa tu instancia real de Supabase desde .env. Inserta datos de prueba y los limpia al final (ver suites en el propio archivo).

node test.js
node test.js --suite formatter,media,connectivity

Claves de suite (entre otras): connectivity, ai, reminders, scheduler, routes, media, …

Solución de problemas

Error al arrancar: supabaseUrl is required

Faltan SUPABASE_URL / SUPABASE_KEY en el entorno, o ejecutas el proceso sin cargar .env (por ejemplo desde otra carpeta).

npm run dev   # desde la raíz del repo

PGRST204 — columna group_name

Ejecuta en Supabase sql/personal_reminders_add_group_name.sql y vuelve a intentar.

Recordatorio no llega

Revisa en Supabase que reminder_time sea anterior o igual a “ahora” cuando el cron corre (zona del servidor) y que status = pending hasta el envío. Mira logs [reminder] y que el proceso Node siga vivo (PUBLIC_URL / monitor). En Twilio Sandbox, el destinatario debe estar unido al sandbox; fuera de sandbox aplican las reglas de la ventana de mensajería de WhatsApp.

No hay respuesta a nota de voz

Esperable sin STT: sin texto en Body el handler retorna al inicio.

Contacto no encontrado

Guarda el contacto antes o revisa nombre/teléfono en contacts.
